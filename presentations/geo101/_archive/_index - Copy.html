Geodata
Geographic data, or Geodata, represents things that have a location.
This can include defined physical or abstract features like
  • roads
  • buildings
  • census boundaries
It can also include temporal or ephemeral events, like
  • 911 calls
  • cloud cover
  • trending, geo-located tweets




onemilliontweetmap.com shows the last one million geo-located tweets in a given geographic extent in real time


There are numerous types of geographic data.
Let's talk about the fundamental data types you can see right in front of you, in OSM.
Geographic data typically is stored and managed in one of two formats:
  • raster data
  • vector data
RASTER
Raster data stores its geographic information in pixels
PIXELS
This information can include color, height, slope, direction, or any other classification or representation of a location you can think of.
CONTINENTS
For the casual user of online maps raster data is encountered in the form of
  • satellite imagery
  • weather data (cloud cover, precipitation)
  • digital elevation model (topography).




Elevation

Raster analysis is very powerful. The format is conducive to complex spatial analyses and modeling:
  • kernel density (a.k.a."heat-mapping")
  • overlay (e.g., site suitability)
  • flow (e.g., surficial hydrology)
In OSM, raster data is provided to you in the form of satellite imagery, which provides you with a context.
Try changing the background imagery to different sources and transparencies.
When creating and editing OSM data, the different imagery layers provide different sources for validation.
Vector
Vector data stores geometry, attribute, and location information
No matter how much you 'zoom in' on vector data, you won't see pixels.
The data is dynamically rendered as the image you see on the screen.
Vector comes (primarily) in three geometric flavors:
The OSM translation:
  • point = point
  • line = line or "way"
  • polygon = "area"
The editing tools for adding these kinds of vector features to OSM are at the top!
If you hover over the point, line, or area editing tools, the tooltip hints that certain kinds of things are best represented by certain vector types.
Select the Map Data button and then Map Features drop-down
Turn some layers on and off!
Select a feature on the map.

Look at its attributes!

Attributes are an important component of vector data.

Think of a single vector feature, like the Carnegie Museum, as a row in a table. Each column in that table is an attribute.
unique feature id  name operator address website
0001  Carnegie Museum of Natural History Carnegie Institute Unknown carnegiemnh.org
0002  another building another operator another address anotherlink.com
A vector dataset usually stores lots of individual features.
You'll notice that if you have selected a building or street, you'll see on the map the feature is shown as series of of connected points and lines.
This is where the actual geometry and geographic referencing is happening.

WHAT DOES IT MEAN
Let's back up.
You've probably seen a some depictions of Earth, which is a lumpy, sphere-like 3-D object, as a map, which is a smooth, planar, 2-D object.
The history and science behind how precisely we get from one to the other is definitely worth another session entirely.
Obviously, we're trying place these features at a location on the earth. But the Earth is not a perfect sphere.

It is more like a lumpy ball.
Determining where things are located on a lumpy ball is difficult. It's much easier if we can approximate location to a smooth ball.
Rasters: Geographic reference information is typically stored in a complementary projection file
data model: nodes / ways / relations

What did we learn today?
OSM is an open-source, global, and editable geodata source that anyone can use and improve.
Geodata are the elements that make up a map.
Raster and Vector are the basic geodata types used in OSM.
Raster data is made of pixels, which is most commonly encountered wit satellite imagery, but can be used to represent many types of things.
Vector data is made of points, lines, and polygons (or "areas" in OSM-speak).
You can download vector data from OpenStreetMap!
There are lots of places on the internet to find geographic data.
Heck yes!
Just knowing about these few data types can make you a very strong geographic data user.
